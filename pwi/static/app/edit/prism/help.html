General Description

The Image module is a search, display and edit page all in one. A user must be logged in to edit but can search without logging in.

Keyboard Short-cuts

There are reported issues using these short-cuts with Mac/Firefox.
  • Add = Ctrl-Alt-A
  • Delete = Ctrl-Alt-D
  • Search = Ctrl-Alt-S
  • Clear = Ctrl-Alt-C
  • Modify = Ctrl-Alt-M
  • First = Ctrl-Alt-F
  • Previous = Ctrl-Alt-P
  • Next = Ctrl-Alt-N
  • Last = Ctrl-Alt-L

Color-coded Labels

  • Thistle = fields that may be searched (but not modified)
  • Green = fields that may be modified by the Curator
  • Blue = Read-only fields
  • Yellow = Action buttons

Supported Browsers

  • Chrome and FireFox are supported on both PC and Mac computers.
  • Safari is NOT supported for the PWI. There are features that do not work.

Searching

All fields that have labels highlighted in thistle/blue/green may be used for searching. They may be searched individually or in combination. To include data in a tab in a search: click on the tab to expand it and enter text to be searched. When more than one field is used in a search, it is treated as an AND search.

Wildcards

  • Wildcards (%) may be used for most fields.
  • A wildcard may be used at the beginning, middle and/or end of your search text: e.g. %paired%, casein% or AK32%43%.

Search by Dates

  • Valid date formats for searching are:
    • mm/dd/yyyy or mm-dd-yyyy
    • mm/dd/yy or mm-dd-yy
    • yyyy/mm/dd or yyyy-mm-dd
  • Searching for a range of dates:
    • equal to: enter a single date
    • greater than: > mm/dd/yyyy
    • greater than or equal to: >= mm/dd/yyyy
    • less than: < mm/dd/yyyy
    • less than or equal to: <= mm/dd/yyyy
    • between two dates (inclusive): mm/dd/yyyy..mm/dd/yyyy

Dialogue warning boxes(popups) behavior

  • When you are deleting images, you will be given a popup verifying that you want to do this.
  • Depending on how quickly you delete and your browser (Firefox in particular), subsequent popups might include a check box asking if you want to "Prevent this page from creating additional dialogs."
  • Do NOT click this because clicking Yes in the box is required to enable deletions (and if the box is not shown, you cannot click Yes).
  • The only way to undo clicking the checkbox is to quit and relaunch the browser.
  • Therefore, it is best to IGNORE this checkbox.

Duplicate

  • This allows you to create a new image stub using values from an existing stub. All fields but External Link Notes and MGI Image/PIX will be copied to the new image stub. Tip: change the figure label to the one you want for the new stub before hitting Duplicate.

    • Deselect

  • If you deselect the highlighted stub in the result return, all fields will be cleared except J# and copyright. This is useful when you have modified an existing stub and want to create a new one associated with that J#.

    • Image Classes and Types

    There are three classes of images:

    • Expression
    • Molecular
    • Phenotypes

    There are two types of images:

    • full size
    • Thumbnail

    Full size phenotype images can be modified to Molecular or Molecular to Phenotype. The type of thumbnail will follow that of the full size image.
    All images contain at least one image pane, whether it is named or not. (Data associations are made to the pane, not the image.)

    Expression Images

    • Only full sized images are created with each add.
    • Expression curatorial policies:
      • GXD routinely enters image stubs without attaching JPGs to them
      • GXD routinely enters image panes.
      • GXD uses Prism, so individual panes may have X,Y coordinates.

    Phenotype/Molecular Images

    • A full size and thumbnail image is created with each add.
    • The default when you enter via MGD images is type = phenotype.
    • Phenotype curatorial polices:
      • Pheno does not enter image stubs without attaching JPGs.
      • Pheno does not create named image panes.
      • Pheno does not use Prism, so individual panes will not have X,Y coordinates.
      • For thumbnail images: the caption is displayed in the FeWI but the copyright field is not, so a copyright statement is not required for the thumbnails

    Copyright

    There are various flavors of copyrights
    1. Journals that have given image permissions specifically to GXD or MGI. The statements are static and haven't changed over time.
    2. Open Access Journals (generally use creative commons licenses). The license has likely changed over time (either type, version, or both)
    3. Hybrid Access Journals (use creative commons, but not for every article). Curators choose either the license or no license.

    Generally the field is autopopulated during J# validation:

    • If there is only 1 copyright for the journal in the Journal Simple Vocab, it will autopopulate.
    • If there is more than 1 copyright for the journal in the simple vocab there are 3 possibilities:
      1. The curator will see a red picklist above the copyright called 'Journal Licenses'. Curator checks the pdf and chooses the correct license.
      2. For journals with a hybrid policy, the curator will see a red picklist. One entry is 'No image permissions'. Curator chooses between a license and no copyright statement.
      3. The database determines which license to use based on the year of publication.

    Other features of the copyrights field.

    • Curator can remove or modify copyrights, but a QC will list J#'s that have more than 1 copyright statement.
    • Some copyrights require a DOI ID. If this is null, a warning to the curator may be returned.
    • A chosen copyright statement will carry into the next figure for a J#, if the curator doesn't clear the module. Otherwise, curator will have to choose the copyright again.

    JPGs

    • JPGs are attached to image stubs using the Image Submission module.
    • JPGs are submitted to PixelDB, creating a PixID. This PixID is then associated with the image stub.
    • To detach a JPG from a stub, use the erase button and hit Modify. Now you can attach another JPG. (The detached original JPG in now an "orphan" in PixelDB; these orphans are periodically purged.)

    X, Y Coordinates

    • The system determines the X,Y coordinates of an attached JPG.
    • The coordinates for an individual pane are entered using Prism.

    External Link Notes

    • Used by phenotypes.

    Other Accession IDs

    • Used by Expression.
    • Currently IDs displayed for: GenePaint, and GUDMAP.
    • (There are also IDs stored for EMAGE but they are not displayed in the PWI as they are associated with panes; the other 3 are associated directly with the stub.)

    Allele Associations

    • These are used by phenotypes.
    • Alleles can be associated with an image stub in two ways; they can be directly indexed to an image or indirectly via a genotype that includes the allele.
    • The allele associations are created/updated only by clicking on the Modify Allele Associations button; they are not changed by modifications saved by using the Modify button in the upper right corner of the form.
    • When the button is clicked, the system ''scans'' the caption field to find the MGI id of alleles within the allele tag ("\AlleleSymbol(MGI:xxxx|0)|).
    • If the MGI id is a valid allele id, it will be added to the Allele Associations table. If it is not a valid id, it will not be added.
    • To change/update an association, the Curator must create/change the caption, hit the Add/Modify button in the upper right hand corner, and then click the Modify Allele Associations button.
    • If there are no alleles IDs in the caption, the allele associations will be removed when you click the button. You need to be careful, though, because allele associations established via indexing to a genotype (eg J:175213) can be lost this way.
    • The default association of an image to an allele is primary=no; to make that stub the primary stub, the curator will have to change the value in the allele module. (There is no requirement to have a primary image; they can all be primary=no.)
    • Molecular images should only have one associated allele; Phenotype images can have many.
