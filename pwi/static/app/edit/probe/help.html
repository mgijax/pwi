P-WI Probe Editor Interface functionality

General Description

The Probe module is the editorial interface for entering, modifying, and searching various types of molecular segments used for GXD assays and cDNA clone loads. A user must be logged in to edit but can search without logging in.

Clear Buttons

  1. Clear All: clears all fields.
  2. Clear Probe Info: clear all except for Segment type and Reference table row 1.
  3. Clear Partial: clear these fields.
    • Name
    • Region Covered
    • Parent Clone info/insert site/insert size
    • Primer sequence 1, 2, product size
    • Accession IDs (all)
    • Aliases
    • Create/Modification user/dates

  • Safari is NOT supported for the PWI. There are features that do not work.

Searching

All fields that have labels highlighted in thistle/green may be used for searching. They may be searched individually or in combination. When more than one field is used in a search, it is treated as an AND search. Unlike in the antibody module, searches in the Probe Name field do not search aliases; aliases must be searched independently. To include data in a note tab (probe note, raw sequence note) in a search: click on the tab to expand it and enter text to be searched.

You must tab out of the marker field and any of the J# fields to search. These fields require validation, otherwise your search will not work and may spin/timeout.
The MGI field search can support MGI:###### or ######. The MGI must be uppercase to work; it is case-sensitive. To search an accession ID type, pick the Type in the picklist and add % in the Accession ID field.

The J# search field at the top of the page:

  • Displays the J# from the first row of the Reference table (not necessarily the same as the marker table J#).
  • Search will return probes where associated reference is in any row of the reference table.
  • You must tab out of the search to work.
  • Enter J:##### or #####.

Wildcards

  • Wildcards (%) may be used for most fields.
  • A wildcard may be used at the beginning, middle, and/or end of your search test: e.g. %paired%, casein%, or AK32%43%.

Searching using the Search Summary button

  • This button can be used with searches that use the following fields:  segment type and name.
  • When you search with those fields and click the button, it will report the search results in a Probe Summary Results page. This summary opens in a new tab, so be sure you have enabled popups for the pwi in your web browser.
  • The “You Searched For” at the top of the Summary will list the search criteria used to generate the report.
  • Although searches using this button may appear to be the same as searching within the module, they use different code.
  • Of special note:
    • Unlike searching within the module, name field searches also include the alias field.
    • If you include fields other than segment type and name in your search, they will be ignored.

Search by Dates

  • Valid date formats for searching are:
    • mm/dd/yyyy or mm-dd-yyyy
    • mm/dd/yy or mm-dd-yy
    • yyyy/mm/dd or yyyy-mm-dd
  • Searching for a range of dates:
    • equal to: enter a single date
    • greater than: > mm/dd/yyyy
    • greater than or equal to: >= mm/dd/yyyy
    • less than: < mm/dd/yyyy
    • less than or equal to: <= mm/dd/yyyy
    • between two dates (inclusive): mm/dd/yyyy..mm/dd/yyyy

Entering Probes

  • Required fields: Segment Type, Name, Reference table J#. Other fields will fill in by default (see Defaults section).
  • Probes can have multiple markers and/or multiple references associated.
  • Each associated reference can have multiple Accession ID's and/or multiple Aliases.
  • Probes can be entered without a marker.
  • Autopopulate cDNA and Autopopulate Not Specified Buttons can be used to fill out the form. See Autopopulate table.

Strain, tissue, and cell line fields

  • The Strain, Tissue, and Cell Line fields use controlled vocabularies.
  • These fields auto complete as you type, so you can choose from the entries.
  • Tab out.
  • If you enter a value for auto complete, and do not tab out of the field, then you have to Clear All twice. This is a browser issue and has to do with the combination of auto complete and validation.
  • If you try to enter a term that does not currently exist, you will be given a dialogue box asking if you want to proceed.

Strain: a list of strains currently in probes and antigens. To see the list go to the "Show JSON package/API domain + other data" yellow button below the probe module fields and search for - "strainlookup"
Tissue: a drop down of tissues currently in the source table aids entry into this field; be sure to tab out to ensure the PWI remembers your choice. To see the list go to the "Show JSON package/API domain + other data" yellow button below the probe module fields and search for - "tissueLookup"
Cell Line: the list of existing cell lines can be found here: Existing Cell lines Report ( PRB_CellLine.sql.rpt )

Entering Primers

  • Spaces, tabs, and line breaks will be removed from the sequence fields. Non-printable characters (from space up to the tilde) are also removed. Slash, parentheses, hyphens and underscores can be used.
  • Autopopulate Primer Button can be used to fill out the form. See Autopopulate table.

Relationship 'R'

This describes the relationship between the molecular segment and the marker.
The Relationships for probes are:

  • E-encodes; is assigned by a curator and is used when the segment comes from a mouse gene. (See below for details regarding the auto-E association.)
  • H-hybridizes; is assigned by a curator and is a weaker relationship than encodes. It means that the segment hydridizes to the genetic marker. This is used
    1. for probes from species other than mouse (or if the species is not specified)
    2. for consensus sequence probes, for example a probe to all homeodomains
    3. or mouse ESTs that blast poorly to the marker.
  • A-amplifies; added by the curator for all primers.
  • P-putative; is computationally assigned to downloaded ESTs in the Unigene load.

Source Defaults

  1. If no Parent Clone is given, create new source record
    • Segment Type = Not Specified
    • Vector Type = Not Specified
    • Organism = Not Specified
    • Strain = Not Specified
  2. If Tissue = Not Specified, then Cell Line = Not Specified, Age = Not Specified
  3. If Tissue = Not Applicable, then Cell Line = Not Applicable, Age = Not Applicable
  4. If the organism is not mouse, laboratory or Not Specified, then the Strain field default should be Not Applicable.
  5. If a tissue is entered, then the Cell Line Field default should be Not Applicable.
  6. If a Cell Line is entered, then the Age default should be Not Applicable.

Auto-E

If a marker and probe both have the same sequence ID associated with them, the database will add an Auto-E association to the probe record. This is a nightly process. J:85324 is the reference used for an auto-E. This is a fairly stringent assignment of relationship but is subject to change if the gene assignment of a sequence changes.

Has Expression?

This indicates the probe/primer is used in an assay. The value cannot be modified by the curator (except by adding or deleting assays).

References with many probes

J:57656 is the IMAGE clone reference with 378,859 probes, and J:82718 is another clone load with 1.6 million probes. If used for searching, combine with other parameters to get a reasonable number of probes returned (e.g. marker).
