P-WI Genotype/DO Annotations Editor Interface functionality

General Description

The Genotype/DO Annotations module is a search, display and edit page all in one. A user must be logged in to edit but can search without logging in.

Keyboard Short-cuts

There are reported issues using these short-cuts with Mac/Firefox.
  • Search = Ctrl-Alt-S
  • Clear = Ctrl-Alt-C
  • Modify = Ctrl-Alt-M
  • Previous = Ctrl-Alt-P
  • Next = Ctrl-Alt-N

Color-coded Labels

  • Thistle = search only
  • Green = search & edit
  • Blue = read only
  • Yellow = current row

Supported Browsers

  • Chrome and FireFox are supported on both PC and Mac computers.
  • Safari is NOT supported for the PWI. There are features that do not work.

Pulldown Menus

Genotype: State, Compound
MP Annot: Qualifier, Evidence, Sex
DO Annot: Qualifier Evidence
GO Annot: Qualifier Evidence

You can always use the pulldown to select the item you want to use for annotation.

Searching

All fields that have labels highlighted in blue/green may be used for searching. They may be searched individually or in combination. To include data in a tab in a search: click on the tab to expand it and enter text to be searched. When more than one field is used in a search, it is treated as an AND search.

Search By Genotype

The Genotype field includes the strain background followed by the allele pairs(e.g. involves: 129 * C57BL/6 Atp13a2,Atp13a2). Be sure to include wildcards when not entering a full genotype statement. The search results will present the genotypes specified by the search criteria. The main portion of the page displays the DO annotations associated with the selected genotype (denoted by a yellow highlight). Use the mouse or First/Next/Previous/Last buttons to change the selected genotype and DO annotation display.)

Qualifier

The 'NOT' qualifier may be associated with any DO term, but should only be used when there was an expectation that the mouse would be a phenotypic model of the disease. All other DO terms default to (none).

Search for Obsolete Terms

Obsolete DO terms may be searched for by checking the box after .Search Obsolete.. Note only .preferred. obsolete DO terms (example: DOID:0050502) can be searched. Non-preferred obsolete DO terms can.t be searched.

Evidence Code

The only evidence code currently used is 'TAS'.

Dialogue warning boxes (popups) behavior

  • 'Cannot save this Annotation if a record is not selected.' Click 'OK' and select a genotype record.
  • 'Invalid DO Acc ID: DOID:DOID:14330' Click 'OK' and correct invalid DO terms.
  • 'Invalid Reference: J:87272f' Click 'OK' and correct invalid J numbers.
  • 'This reference is not associated to all Alleles of this Genotype. Do you want the system to add a 'Used-FC' reference association for these Alleles?'To add the J number to the alleles associated with the selected genotype, click 'OK'. To prevent the addition, click 'cancel' and edit J number.

    Entering DO Annotations

    Search for relevant genotype(s) by MGI ID or genotype (remember to use wildcards when incomplete information is entered). Paste DO term number (e.g. DOID:14330). Qualifier defaults to 'none' for all terms. Enter J number (J:123456). Tabbing through the field or selecting the next DO annotation row will autofill the J number field with the J# of the previous DO annotation row. The evidence code defaults to TAS when annotation is saved. Either use the Save annotations button below the DO annotation field or below the search and clear buttons.

    Entering Notes

    Select a DO annotation row and add a row in the Notes section below the DO annotation table. Note type defaults to General and that is currently the only note type we use. Notes for DO terms are PRIVATE. Use the 'Save Notes' or 'Save Annotations' buttons to save.

    Modifying DO Annotations

    If there are multiple rows with annotations to the same DO term you can't modify one row without also modifying the other(s). If you want to modify just one row, you need to add a new row, enter the new annotation, and then remove the old annotation.

    Wildcards

    • Wildcards (%) may be used for most fields.
    • A wildcard may be used at the beginning, middle and/or end of your search text: e.g. %paired%, casein% or AK32%43%.

    Search by Dates

    • Valid date formats for searching are:
      • mm/dd/yyyy or mm-dd-yyyy
      • mm/dd/yy or mm-dd-yy
      • yyyy/mm/dd or yyyy-mm-dd
    • Searching for a range of dates:
      • equal to: enter a single date
      • greater than: > mm/dd/yyyy
      • greater than or equal to: >= mm/dd/yyyy
      • less than: < mm/dd/yyyy
      • less than or equal to: <= mm/dd/yyyy
      • between two dates (inclusive): mm/dd/yyyy..mm/dd/yyyy
