Experiment Curation

Keyboard Shortcuts
  • Search = Ctrl/Alt S
  • Clear = Ctrl/Alt C
  • Modify = Ctrl/Alt M
  • Previous = Ctrl/Alt P
  • Next = Ctrl/Alt N
  • Last = Ctrl/Alt L
Color-coded Labels
  • Blue/Green = fields that may be searched (but not modified).
  • Green = fields that may be searched or modified by the Curator.
  • Blue = read only fields
  • Yellow = action buttons
Curation Status Fields
  • Evaluation = The primary curated field for Experiments. The GXD Curator will determine whether or not they want to curate the Samples of this Experiment (Yes, No, Maybe). This field is not displayed anywhere and does not determine whether Experiments are moved to Public.
  • Curation State = This field is set by the software and cannot be modified by the Curator. Experiments with a Curation State = Done are moved to the Public WI. Once an Experiment has curated samples -- the Curation State is updated to Done. Until there is an ability to delete Samples (future enhancement) a Curator is unable to revert the state to "Not Done". Once the Curation State = Done the value of Evaluation State is irrelevant.
ID Fields for an Experiment
  • ArrayExp = experiment ID from ArrayExpress; you may link to this ID using the icon to the right of the input box.
  • GEO = experiment ID from GEO; you may link to this ID using the icon to the right of the input box.
  • PubMed = publication ID; you may click on this ID to link to PubMed; there may be 0-n PubMed IDs and this count is displayed next to the label. "Pub Med: n"
Curated Experiment Fields (Green)
  • Title = A Curator may modify the Title; likely will be limited to infrequent typo corrections as this field will be displayed in the Public WI.
  • Description = A Curator may modify the Description; likely will be limited to infrequent typo corrections as this field will be displayed in the Public WI.
  • Experiment Note = A text field used at the discretion of the Curator. This text is displayed in the WI when present.
  • Experiment Type = A Curator may modify the Experiment Type. The value displayed in the "Green" Experiment Type is the one that the Load set. The value(s) displayed in the "Grey" Experiment Type are the ones downloaded from ArrayExpress.
  • Study Type = A Curator selects the appropriate Study Type of the Experiment when curating the Sample Data. The load sets this field to "Not Curated".
  • Experimental Variables = The items listed are defined in a Simple Vocabulary, the abbreviations are used to save space. A Curator may select 0-to-many terms. The load sets this field to "Not Curated".
Date Fields
  • Creation in GXD = Date this experiment was loaded in the MGD database.
  • Release = Date set by ArrayExpress
  • Last update = Date set by ArrayExpress
  • Evaluated = Most recent date that the Evaluation Status was modified. Modifications to any other curated Experiment fields not tracked by this date field (e.g. Title, Study Type, etc).
  • Initial Curation = Date the first curation of this Experiment's Samples occurs.
  • Last Curation = Most recent date that the curated Sample Data was modified.
Search by Dates (Creation in GXD, Release, Evaluated, Initial Curation, etc)
  • Valid date formats:
    • mm/dd/yyyy or mm-dd-yyyy
    • mm/dd/yy or mm-dd-yy
    • yyyy/mm/dd or yyyy-mm-dd
  • Searching for a range of dates:
    • equal to: enter a single date
    • greater than: > mm/dd/yyyy
    • greater than or equal to: >= mm/dd/yyyy
    • less than: < mm/dd/yyyy
    • less than or equal to: <= mm/dd/yyyy
    • between two dates (inclusive): mm/dd/yyyy..mm/dd/yyyy
Search with Wildcards
  • The wildcard character is %.
  • You may append or prepend the wildcard (%) to your search text.
  • A wildcard may be used in text, date, and ID searches.

Samples - Raw & Curated

Raw Samples: Download sample metadata
  • A raw sample table is downloaded by clicking 1 of 2 links - with or without consolidation. The number of columns and content will vary between experiments -- the columns are named and defined by the submitters of the data.
  • Not all Experiments have Samples that require consolidation. Counts are displayed at the top of the raw sample table that informs the Curator whether or not consolidation has occurred.
  • If required, consolidation is done most commonly by Source Name. In the rare cases of Samples not having a Source Name the Assay Name will be used. If there is no Assay Name, then Extract Name is used for consolidation.
  • Consolidated Samples with multiple (non-duplicated) values will be delimited by pipes (|).
  • If a Curator determines that the samples should not be consolidated for curation they should redisplay the experiment and click on "Download sample metadata no consolidation".
  • Downloading sample metadata after samples have been curated...
Curate Samples
  • Curation of Samples happens when 2 actions occur: clicking on the "Curate Samples" link and clicking on "Modify". Once these actions are taken this experiment and samples will be moved to the Public WI.
  • Clicking on the Curate Samples link displays a table of curated sample fields. A curator may alter the default display by checking/unchecking boxes. These selections will be maintained until another experiment is selected.
  • If a Curator selects a different experiment before saving one that is in progress -- no samples are saved.
Sample Fields & Validation
  • Name: default most commonly the Source Name; may be Assay Name or Extract Name if Source Name does not exist; Curator may reenter this field if desired.
  • Organism: selection from pull-down list of controlled vocabulary
  • GXD Relevant?: selection from pull-down of controlled vocabulary
  • Genotype: Genotype ID entered with or without "MGI:"; validation and display of genotype when clicking or tabbing out.
  • Age Unit: selection from pull-down of controlled vocabulary
  • Age Range: free-form text field; valid values differ by Age Unit; Warning in place for alpha entry.
  • Sex: selection from pull-down of controlled vocabulary
  • EMAPS: term selected from the clipboard or ID entered. Validation performed when tabbing out. Reset to blank by clicking out.
  • Note: optional free form text.
UI features
  • Toggle columns on/off: available for curated and raw columns either by check box or X at top of column.
  • Column sort: available for curated and raw columns
  • Filters: available for raw columns only
  • Clipboard: updates to the EMAPA clipboard are recognized once the Refresh Clipboard link is clicked.
  • Copy Up/Down: available for curated columns; clicking on an arrow replicates the value from that cell for all cells above or below (depending on which arrow used).
  • Copy from Above: available for curated columns; tabbing out of a field will pull in the value from a cell above.

GXD HT Experiment Load Overview

This is an incremental load of high-throughput experiments from ArrayExpress. These are being loaded for the purpose of indexing some of them using standard vocabularies for the meta data of the experiment. It is expected that users will query GXD to find interesting experiments, but will go to the source for the details of the experiment itself.

  • Only experiments for mus and select assay types are loaded.
  • The load runs on a regular weekly schedule.
  • Most of the newly loaded experiments will be set to an Evaluation Status of "Not Evaluated". The exceptions are Experiments described as "SuperSeries", those are set to "No" and Experiments with a PubMed ID in MasterBib, those are set to "Maybe".
  • GXD Curators will triage the newly loaded experiments to determine if the experiment should be indexed (Yes, No, Maybe).
  • At this point there are no plans to delete any of the loaded experiments even if Curators flag them with a "No" (i.e. don't index this one).
  • Once loaded into the MGI database experiments are not updated via the Experiment Load with one exception: the PubMed ID field may be updated if new values are detected at ArrayExpress or GEO.
