GXD High-throughput Evaluation

Keyboard Shortcuts
  • Search = Ctrl/Shift S
  • Clear = Ctrl/Shift C
  • Modify = Ctrl/Shift M
  • Previous = Ctrl/Shift P or Left arrow
  • Next = Ctrl/Shift N or Right arrow
Color-coded Labels
  • Orange = fields that may be searched (but not modified).
  • Green = fields that may be searched or modified by the Curator.
  • Grey = read only fields
  • Blue = action buttons
ID Fields for an Experiment
  • ArrayExp = experiment ID from ArrayExpress; you may link to this ID using the icon to the right of the input box.
  • GEO = experiment ID from GEO; you may link to this ID using the icon to the right of the input box.
  • PubMed = publication ID; you may click on this ID to link to PubMed; there may be 0-n PubMed IDs and this count is displayed next to the label. "Pub Med: n"
Curated Experiment Fields (Green)
  • Title = A curator may modify the Title; likely will be limited to infrequent typo corrections as this field will be displayed in the Public WI.
  • Description = A curator may modify the Description; likely will be limited to infrequent typo corrections as this field will be displayed in the Public WI.
  • Evaluation = The primary curated field for Experiments. The GXD curator will determine whether or not they want to curate the Samples of this Experiment (Yes, No, Maybe).
  • Experiment Note = tbd
  • Experiment Type = A curator may modify the Experiment Type. The value displayed in the "Green" Experiment Type is the one that the Load set. The value(s) displayed in the "Grey" Experiment Type are the ones downloaded from ArrayExpress.
  • Study Type = A curator selects the appropriate Study Type of the Experiment when curating the Sample Data. The load sets this field to "Not Curated".
  • Experimental Variables = The items listed are defined in a Simple Vocabulary, the abbreviations are used to save space. A curator may select 0-to-many terms. The load sets this field to "Not Curated".
Date Fields
  • Creation in GXD = Date this experiment was loaded in the MGD database.
  • Release = Date set by ArrayExpress
  • Last update = Date set by ArrayExpress
  • Evaluated = Most recent date that the Evaluation Status was modified. Modifications to any other curated Experiment fields not tracked by this date field (e.g. Title, Study Type, etc).
  • Initial Curation = Date the first curation of this Experiment's Samples occurs.
  • Last Curation = Most recent date that the curated Sample Data was modified.
Search by Dates (Creation in GXD, Release, Evaluated, Initial Curation, etc)
  • Valid date formats:
    • mm/dd/yyyy or mm-dd-yyyy
    • mm/dd/yy or mm-dd-yy
    • yyyy/mm/dd or yyyy-mm-dd
  • Searching for a range of dates:
    • equal to: enter a single date
    • greater than: > mm/dd/yyyy
    • greater than or equal to: >= mm/dd/yyyy
    • less than: < mm/dd/yyyy
    • less than or equal to: <= mm/dd/yyyy
    • between two dates (inclusive): mm/dd/yyyy..mm/dd/yyyy
Search with Wildcards
  • The wildcard character is %.
  • You may append or prepend the wildcard (%) to your search text.
  • A wildcard may be used in text, date, and ID searches.
GXD HT Experiment Load Overview

This is an incremental load of high-throughput experiments from ArrayExpress. These are being loaded for the purpose of indexing some of them using standard vocabularies for the meta data of the experiment. It is expected that users will query GXD to find interesting experiments, but will go to the source for the details of the experiment itself.

  • Only experiments for mouse are loaded.
  • The load runs on a regular schedule (TBD - monthly/weekly?).
  • Most of the newly loaded experiments will be set to an Evaluation Status of "Not Evaluated". The exception are Experiments described as "SuperSeries", those are set to "No".
  • GXD Curators will triage the newly loaded experiments to determine if the experiment should be indexed (Yes, No, Maybe).
  • At this point there are no plans to delete any of the loaded experiments even if Curators flag them with a "No" (i.e. don't index this one).
