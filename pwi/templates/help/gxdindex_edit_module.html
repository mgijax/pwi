GXD Index EI Functionality


Entering Data:
  • J# field - acceptable formats for the field e.g. J:123456,J123456,j:123456,j123456, or 123456. Tabbing out triggers a validation of the J# and fills in the citation.
  • Marker field - case insensitive. Tabbing out triggers a validation of the gene symbol.
  • Click boxes in table to add an X
  • For first entry of a J#, choose priority (and conditional if different from ‘Not Applicable’ default option). Subsequent entries do not need to choose. If the priority or conditional choices are changed in subsequent entries, the changes are propagated to all entries for the J#.
  • Click Add or Ctrl-Shift-A
  • The record appears in the search results box. Current J# and search results box remain, but the rest of the field is blanked. The cursor focus is in the gene field.
Add copy:
  • Select existing record
  • Make changes to record (at least gene)
  • Click Add (no Copy) or Ctrl-Shift-A
  • The record appears in the search results box (with the new gene). Current J# and search results box remain, but the rest of the field is blanked. The cursor foucus is in the gene field.
Searching:

All fields are searchable individually or in combination. Multiple field entries are an AND search (including searching by multiple ages in the grid). % wild card is used.

  • Marker
  • J#
  • Citation
  • Notes
  • Priority and Conditional menus
  • Coded?
  • Created By and Modified By
  • Date (created by and modified by)
  • Table entries (an X in the table)

Search by Date (created by, modified by)

  • Valid date formats:
    • mm/dd/yyyy or mm-dd-yyyy
    • mm/dd/yy or mm-dd-yy
    • yyyy/mm/dd or yyyy-mm-dd
  • Searching for a range of dates:
    • equal to: enter a single date
    • greater than: > mm/dd/yyyy
    • greater than or equal to: >= mm/dd/yyyy
    • less than: < mm/dd/yyyy
    • less than or equal to: <= mm/dd/yyyy
    • between two dates (inclusive): mm/dd/yyyy..mm/dd/yyyy
Deselecting a record:
  • Clicking on a record that is the current selection will deselect that record
  • This removes the connection between form and search result
  • Clears stages grid
  • Clears marker field
  • Focuses browser on marker input field
Keyboard shortcuts:

Ctrl-Shift-c – clears form, table and search results. Cursor focus goes to J# (no entry).
Ctrl-Shift-a – adds record to the bottom of the search results. Clears form except for J#.
Ctrl-Shift-m – modify. Updates record. No clear.
Ctrl-Shift-d – delete. Removes record. Clears form. Search results remain minus deleted record.
Ctrl-Shift-n – go to next record of search results list. If cursor focus is on a blank form (as in after an add), goes to first entry of search results list.
Ctrl-Shift-p – go to previous record of search results list.
Ctrl-Shift-b – go to bottom record of search results list.

Clearing:

  • Upon hitting clear or Ctrl-shift-c, all fields of form, the search results, and created/modified section are cleared. Focus returns to the J# field, which is blank.
  • Notes field has an X that allows the form to be cleared.

Search results box:

Entries have the gene, J#, first author, short citation. There are buttons to move around in the table, as well as keyboard shortcuts. Arrow keys move around table?
Sort for the results is by gene then by first author/short citation.

Error Messages, Messages, Defaults:
  • An error message and inability to add duplicate records.
  • Message and no entry if you haven’t chosen a priority for the first record of a J#
  • Which Marker? Widget appears in field if you've chosen a marker that has more than one possible match (e.g. because of a withdrawn symbol). Examples: th, T, Pdx1. Use up and down arrow and hit enter, or click the correct marker. Protein coding current symbols are first option.
  • An error message if no stages have been selected for this record.
  • 'Not found' errors for invalid gene or J#
  • 'Selected Withdrawn marker' error (e.g. Rnasen). Current symbol listed in error message.
  • Are you sure you want to delete a record?
  • Defaults when adding a new Index record:
    1. If an Index record exists for the same J:, then use the same Priority for the new Index record
      Else use the Curator-entered value
      Else error, if no value is entered
    2. If an Index record exists for the same J:, then use the same Conditional Mutant for the new Index record
      Else use the Curator-entered value
      Else use ‘Not Applicable’, if no value is entered
