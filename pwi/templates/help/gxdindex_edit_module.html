GXD Index EI Functionality


Entering Data:
  • J# field - acceptable formats for the field e.g. J:123456, j:123456, or 123456. Tabbing out triggers a validation of the J# and fills in the citation.
  • Marker field - case sensitive (ie T and t are treated differently). Tabbing out triggers a validation of the gene symbol.
  • Click boxes in table to add an X
  • For first entry of a J#, choose priority (and conditional if different from 'Not Applicable' default option). Subsequent entries do not need to choose. If the priority or conditional choices are changed in subsequent entries, the changes are propagated to all entries for the J#.
  • Click Add or Ctrl-Alt-A
  • The record appears in the search results box. Current J# and search results box remain, but the rest of the field is blanked. The cursor focus is in the gene field.
Add copy:
  • Select existing record
  • Make changes to record (at least gene)
  • Click Add or Ctrl-Alt-A
  • The record appears in the search results box (with the new gene). Current J# and search results box remain, but the rest of the field is blanked. The cursor focus is in the gene field.
Deselecting:
  • If you deselect an index record in the search return, the J# will be retained. All other fields are cleared, allowing for entry of the next index record.
Searching:

Most fields are searchable individually or in combination. Multiple field entries are an AND search (including searching by multiple ages in the grid). % wild card is used. The search results will list the # of index records matching the search.

  • J#
  • Marker
  • Notes
  • Priority and Conditional menus
  • Coded?
  • Created By and Modified By
  • Date (created by and modified by)
  • Table entries (an X in the table)

Search by Date (created by, modified by)

  • Valid date formats:
    • mm/dd/yyyy or mm-dd-yyyy
    • mm/dd/yy or mm-dd-yy
    • yyyy/mm/dd or yyyy-mm-dd
  • Searching for a range of dates:
    • equal to: enter a single date
    • greater than: > mm/dd/yyyy
    • greater than or equal to: >= mm/dd/yyyy
    • less than: < mm/dd/yyyy
    • less than or equal to: <= mm/dd/yyyy
    • between two dates (inclusive): mm/dd/yyyy..mm/dd/yyyy
Deselecting a record:
  • Clicking on a record that is the current selection will deselect that record
  • This removes the connection between form and search result
  • Clears stages grid
  • Clears marker field
  • Focuses browser on marker input field
Keyboard shortcuts:

Ctrl-Alt-C clears form, table and search results. Cursor focus goes to J# (no entry).
Ctrl-Alt-S search. Performs search.
Ctrl-Alt-A adds record to the bottom of the search results. Clears form except for J#.
Ctrl-Alt-M modify. Updates record. No clear.
Ctrl-Alt-D delete. Removes record. Clears form. Search results remain minus deleted record.
Ctrl-Alt-N go to next record of search results list. If cursor focus is on a blank form (as in after an add), goes to first entry of search results list.
Ctrl-Alt-P go to previous record of search results list.
Ctrl-Alt-L go to bottom record of search results list.

Clearing:

  • Upon hitting clear or Ctrl-Alt-C, all fields of form, the search results, and created/modified section are cleared. Focus returns to the J# field, which is blank.
  • Notes field has an X that allows the form to be cleared.

Search results box:

Entries have the gene, J#, first author, short citation. There are buttons to move around in the table, as well as keyboard shortcuts.
Sort for the results is by gene then by first author/short citation.

Error Messages, Messages, Defaults:
  • A postgres error message and inability to add duplicate records.
  • 'Priority is required' message and no entry if you have not chosen a priority for the first record of a J#.
  • 'At least one Stage is required' message and no entry if no stages have been selected for a record.
  • 'Invalid Reference' error for invalid J#.
  • 'Invalid Marker Symbol' error for invalid or withdrawn symbols.
  • Are you sure you want to delete a record?
  • Defaults when adding a new Index record:
    1. If an Index record exists for the same J:, then use the same Priority for the new Index record
      Else use the Curator-entered value
      Else error if no value is entered
    2. If an Index record exists for the same J:, then use the same Conditional Mutant for the new Index record
      Else use the Curator-entered value
      Else use 'Not Applicable' if no value is entered
Table Layout

A grid slider is available above and below the grid. In addition, 'Early' and 'Late' buttons shift the grid to the 0.5 or A ages, respectively.

Assays are listed in the following order on the left side of the table.

  • Prot-sxn
  • RNA-sxn
  • Prot-WM
  • RNA-WM
  • Knock in
  • Northern
  • Western
  • RT-PCR
  • cDNA
  • RNAse prot
  • S1 nuc
  • Primer ex

Colors for field headers in RGB values:

  • Edit,search,view - Green 195:246:190
  • View only - blue 203:232:248
  • Search,view - thistle 216:191:216

The rows are shaded in sets of three.

General Info, Tab behavior:
  • The # of records field is included for SEs.
  • Tab moves between J#, marker, picklist, notes, priority, conditional, created by, created by date, modified by, modified by date, then back to J#.
