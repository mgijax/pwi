P-WI Simple Vocabulary Editor Interface functionality

General Description

The Simple Vocabulary module is a display and edit page all in one. A search returns all vocabularies. You may not create a vocabulary from this module. You may only add/edit/delete terms. A user must be logged in to edit but can search without logging in.

  • Green = edit
  • Blue = read only
  • Yellow = current row

Supported Browsers

  • Chrome and FireFox are supported on both PC and Mac computers.
  • Safari is NOT supported for the PWI. There are features that do not work.

Searching

There is no searching by Vocabulary/Term attributes. The Search button simply returns all vocabularies.

Wildcards

  • No searching, no wildcards

Journal

Fields
  • Term = Journal name
  • Abbreviation = license used for the copyright statement in the Image Module
  • Definition = used for information or special formatting

Journals can have single or multiple rows.

Definition options
  1. By special permission - journal-specific permission given to GXD or MGI.
  2. Pop-up text - shorthand used in the image module popup picklist for curator to choose the correct license. e.g. CCBY-4.0
  3. SQL format used by Image module during J# validation to choose appropriate license to autopopulate based on year of publication. No other text permitted.
  4. No image permissions - entered when abbreviation field is empty. In the image module popup, this entry gives curators the option to not fill in the copyright when no license allows.

List is automatically re-ordered (alpha-numeric) by Term (journal name) after Save.

SQL formats allowed:
SQL(<2008)
SQL(<=2008)
SQL(>2008)
SQL(>=2008)
SQL(=2008)
SQL(between 2003 and 2008)
Many journals have multiple rows because the copyright license changed over time. This notation indicates the year range for each copyright. It informs the image module copyright field of the appropriate abbreviation to enter.

License formating:

  • Asterisk is used to mark the place for entering the short citation. (e.g. Biol Open)
  • Links are written in html. (e.g. Creative Commons License e.g. BMC Biol)
  • DOI ID's inserted at || (e.g. Dev Biol). This must be coded behind the scenes by SE
  • Second asterisk inserts the year of publication (e.g. PNAS). This must be coded behind the scenes by SE

Standard format of copyrights:

  • This image is from *, and is displayed with the permission of Publisher Name(link or not) who owns the copyright.
  • This image is from *, and is displayed displayed under the terms of the {A HREF=http://creativecommons.org/licenses/by/4.0/}Creative Commons Attribution 4.0 International License{/A}.
    • Replace brackets with angle brackets.
