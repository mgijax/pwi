P-WI Strain Editor Interface functionality

General Description

The Strain module is a search, display and edit page all in one. A user must be logged in to edit but can search without logging in.

Keyboard Short-cuts

There are reported issues using these short-cuts with Mac/Firefox.
  • Search = Ctrl-Alt-S
  • Clear All = Ctrl-Alt-C
  • Modify = Ctrl-Alt-M
  • Previous = Ctrl-Alt-P
  • Next = Ctrl-Alt-N

Color-coded Labels

  • Thistle = search only
  • Green = search & edit
  • Blue = read only
  • Yellow = current row

Supported Browsers

  • Chrome and FireFox are supported on both PC and Mac computers.
  • Safari is NOT supported for the PWI. There are features that do not work.

Searching

All fields that have labels highlighted in blue/green may be used for searching. They may be searched individually or in combination. To include data in a tab in a search: click on the tab to expand it and enter text to be searched. When more than one field is used in a search, it is treated as an AND search.

Search by Dates

  • Valid date formats for searching are:
    • mm/dd/yyyy or mm-dd-yyyy
    • mm/dd/yy or mm-dd-yy
    • yyyy/mm/dd or yyyy-mm-dd
  • Searching for a range of dates:
    • equal to: enter a single date
    • greater than: > mm/dd/yyyy
    • greater than or equal to: >= mm/dd/yyyy
    • less than: < mm/dd/yyyy
    • less than or equal to: <= mm/dd/yyyy
    • between two dates (inclusive): mm/dd/yyyy..mm/dd/yyyy

Search Data

When you click on the Search button:

  • the results are displayed in a table with each row representing a single allele.
  • results are displayed in alpha-numeric order.

Find Duplicate Strain

Use the ‘Find Duplicate’ button or search for duplicate strains using the strain symbol field and ‘Search’ button.

Deleting Existing Strain

Strains cannot be deleted if associated with:

  • Allele
  • Allele Cell Line
  • Allele Variant
  • Genotype
  • Mapping/FISH
  • Mapping/In Situ
  • Mapping/RI Set

    • Strains can be deleted if not associated with the above but are associated with:

  • Image Pane
  • Mapping Coordinates
  • Notes
  • Reference Associations
  • Synonyms
  • Strain/Genotype Assoc
  • Strain/Marker Assoc

    • Merge Strains

    Here are the merge rules:

    • Cannot merge Private Strain into Public Strain.
    • Cannot merge Private Strain into Public Strain.
    • Cannot merge Public Strain into Private Strain
    • Cannot merge Standard Strain into Non-Standard Strain
    • This merge would create a duplicate entry for Probe XXXXX
    • Incorrect and Correct Strains must have the same Marker Symbols
    • Incorrect and Correct Strains must have the same Strain Attributes
    • Incorrect and Correct Strains must have the same Species
    • Incorrect and Correct Strains must have the same Needs Review values
    • Incorrect and Correct Strains must have the same JAX Registry Number.
    • Move all MGI:xxx of Incorrect Strain to Correct Strain/other MGI id
    • Remove any Accession records belonging to incorrect strain which already exist for correct strain (i.e. remove duplicate accession ids)
    • Transfer:  Allele, Allele/CellLine, Strain/Marker, Synonyms, Genotype, Notes
    • Transfer:  Probe Source, Probe/Allele/Strain, MLD/FISH, MLD/InSitu, Cross, RI
    • It does *not* transfer:  Strain/Reference relationships 
    • It does *not* transfer:  Strain/Genotype relationships

    For the notes fields (Public Note, Strain Origin, IMPC Colony ID, and Mutant cell Line of Origin)

    • If both incorrect and correct strains have notes, concatenate incorrect notes onto correct notes
    • Else if only incorrect strain has notes, move incorrect notes to correct notes
    • Else if only correct strain has notes, do nothing
    • If fields are identical, then do nothing
