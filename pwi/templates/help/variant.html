P-WI Edit Variant functionality:

General Description

The Variant module is a search, edit, display page all in one. Each variant is related to a single allele, while an allele can have multiple variants. Detail for these variants may be viewed and edited one at a time (not all variants at once for a given allele).

Keyboard Short-cuts

There are reported issues using these short-cuts with Mac/Firefox.
  • Search = Ctrl/Alt S
  • Clear = Ctrl/Alt C
  • Modify = Ctrl/Alt M
  • Previous = Ctrl/Alt P
  • Next = Ctrl/Alt N
  • Associate Tag = Ctrl/Alt T

Color-coded Labels

  • Blue/Green = fields that may be searched (but not modified)
  • Green = fields that may be modified by the Curator
  • Blue = Read only fields
  • Yellow = Action buttons

Supported Browsers

  • Chrome and FireFox are supported on both PC and Mac computers.
  • Safari is NOT supported for the PWI. There are features that do not work.

Searching

All fields that have labels highlighted in blue/green and may be used for searching. They may be searched individually or in combination. At least 1 search field must be entered or an error message is displayed. When more than 1 field is used in a search, it is treated as an AND search. (i.e.- All entered fields must match the variants to be returned.)

Limit on # of results
  • The Variant module is currently configured to restrict the number of results to 1,001 variants. It may be preferable to use more search fields to reduce the number of results returned.
Wildcards
  • Wildcards may be used for most fields. Exceptions include: Accession IDs, Creation Date, and Modification Date.
  • A wildcard (%) may be used at the beginning and/or end of your search text. E.g. %paired% or casein%
Search by Accession IDs
  • Valid types of IDs: MGI Allele
  • A single or list of IDs may be entered. The IDs may be of one type or a mix.
  • Lists may be delimited by comma, space, or newline/carriage return
  • A list of IDs may be entered using Cut/Paste from an Excel file or QC report.
Search by Dates
  • The Creation and Modification Dates are time stamps held in the database.
  • Valid date formats for searching are:
    • mm/dd/yyyy or mm-dd-yyyy
    • mm/dd/yy or mm-dd-yy
    • yyyy/mm/dd or yyyy-mm-dd
  • Searching for a range of dates:
    • equal to: enter a single date
    • greater than: > mm/dd/yyyy
    • greater than or equal to: >= mm/dd/yyyy
    • less than: < mm/dd/yyyy
    • less than or equal to: <= mm/dd/yyyy
    • between two dates (inclusive): mm/dd/yyyy..mm/dd/yyyy

Search Results

When you click on the Search button, the results are displayed in a table with each row representing a single variant. The table displays the variants, through the alleles they are attached to, in alpha numeric order. The variant highlighted in pale yellow(by default the first one) is the one that is displayed in detail in the main form. Youcan highlight a different variant by clicking on a different row.

Adding New Data


  • To create a new variant from scratch, start with a blank form (click Clear if necessary) and enter either an Allele ID or Allele Symbol in their respective fields. Then either hit the TAB key or use the mouse to click out of the field to any other field. The missing allele value (either ID or Symbol) plus associated data (References, Chromosome, Strand) will now be retrieved and filled in. Allele data can now be filled in (see “Sourced and Curated Records Requirements” below for requirements). A variant record will only be created after clicking Create. Subsequent changes are saved by clicking Modify (the Create button will be grayed out once the record has been created).
  • To create a variant that is a copy of (or very similar to) an existing variant, but has to be attached to a different allele of the same gene, click the Duplicate Variant button on the form of the variant record you want to duplicate. This will create a new form with all the variant data of the original form but with only the allele gene symbol filled in for the allele data. Complete the Allele Symbol to represent the new allele you want the variant to be attached to. Then either hit the TAB key or use the mouse to click out of the field to any other field. The missing allele value (Allele ID) plus associated data (References, Chromosome, Strand) will now be retrieved and filled in. Modify variant data as necessary. A variant record will only be created after clicking Create. Subsequent changes are saved by clicking Modify (the Create button will be grayed out once the record has been created).
  • To create a new variant for an existing allele (i.e. you have an allele variant record open), click the Duplicate Allele button. A new form will be created with just the allele data from the previous form filled in; the variant fields will be empty. Allele data can now be filled in (see “Sourced and Curated Records Requirements” below for requirements). A variant record will only be created after clicking Create. Subsequent changes are saved by clicking Modify (the Create button will be grayed out once the record has been created). Once the new record has been created the ‘All variants’ table at the bottom will be updated with the new row of variant data added to the end of the table.
  • If you only provide Sourced info, then both Sourced and Curated records will be created but the Curated record will be empty; if you provide both Sourced and Curated info, then both records will be created and populated using Sourced and Curated info. o This means Sourced and Curated records will always be create regardless of whether you provide only Sourced or only Curated data, or both.

    • Modifying fields
    • All fields with a green label tab (except Types and Effects) accept keyboard or copy-pasted input.
    • To copy values from the As Sourced fields to the corresponding Curated fields, you can click the Copy all button to copy all data or click the Coords or Alleles button in the Genomic, Transcript or Polypeptide sections to copy only selected data. The Coords button copies only the coordinate data (Genome Build/Transcript ID/Polypeptide ID, Start, End) in that section to the corresponding Curated section; The

      Alleles

      button copies only the Reference and Variant Allele data (sequence) in that section to the corresponding Curated section.
    • To enter amino acid sequence you can click the Amino Acid Popup. This will open a popup window with all the valid amino acids and their three-letter and one-letter codes. You can use this window to enter amino acids or use it just as a reference when using manual keyboard entry. If the former, click on any name or code to copy that value to the field that is active (i.e. with a pale yellow background). Note that the system will interpret full names and three-letter codes as strings of single-letter amino acid codes and, upon saving, capitalize all characters. So make sure that in the Polypeptide Reference and Variant Allele fields you only use single-letter codes (upper or lower case). Full names or three-letter codes can be used in either of the Notes fields.

    Allele Section

    • All fields are searchable.
    • A wildcard is allowed in the Allele Symbol field for searching.
    • Allele References are displayed as a comma-separated list.
    • The Strand field will have a red background when the strand is "-".

    Sourced and Curated Records Section Requirements

    Genomic
    • Genome Build must exist if you have Start and Stop coordinates.
    • Genome Start coordinate must be less than or equal to the Stop coordinate.
    • Genome coordinates must have both Start and Stop, not just one of them.
    • You can enter nucleotide sequence in either upper or lower case; lower case letters are automatically capitalized upon saving.
    Transcript
    • If a transcript sequence has a Start and Stop coordinate it must have an ID (which will be validated).
    • Valid Transcript IDs are RefSeq, Ensembl, or Genbank sequences. Use only unversioned IDs(i.e. omit the dot-version-number after the primary ID).
    • A transcript Stop coordinate must be less than or equal to the Start coordinate.
    • You can enter nucleotide sequence in either upper or lower case; lower case letters are automatically capitalized upon saving.
    Polypeptide
    • If a polypeptide sequence has a Start and Stop coordinate it must have an ID (which will be validated).
    • Valid Polypeptide IDs are RefSeq, Ensembl, Swiss-Prot/UniProt, or TrEMBL sequences. Use only unversioned IDs(i.e. omit the dot-version-number after the primary ID).
    • A polypeptide Stop coordinate bust be the same or greater than the Start coordinate.
    • You can enter amino acid sequence in either upper or lower case; lower case letters are automatically capitalized upon saving.

    HGVS Notation

    • Displays the Human Genome Variation Society notation for the variant.

    Variant Reference

    • The reference(s) from which the variant is/are sourced. This field can have one or more references.
    • The reference J numbers are validated.

    Public and Private Notes

    • Public Notes are used to communicate information about the variant to public users.
    • Curator Notes are private and used to document information obtained from the curation process.

    Variant Types and Effects

    • Click the

      Types Popup

      link to bring up a popup window with a selection of variant types. Click the check boxes in front of the types you want to apply to the variant and click Save Changes and Close to save the selection. The selected types will be filled in in the Types field. Alternatively, click Discard Changes and Close (or the window close button top right) if you do not want to save the selection. If there is existing data in the Types field that selection will show up as selected in the popup window. Any modifications to existing selections will be reflected in the field after save and close. A variant can be of more than one type. Type SO terms are shown with the most common first, logically grouped.
    • Click the

      Effects Popup

      link to bring up a popup window with a selection of variant effects. Click the check boxes in front of the effects you want to apply to the variant and click Save Changes and Close to save the selection. The selected effects will be filled in in the Effects field. Alternatively, click Discard Changes and Close (or the window close button top right) if you do not want to save the selection. If there is existing data in the Effects field that selection will show up as selected in the popup window. Any modifications to existing selections will be reflected in the field after save and close. A variant can have more than one effect. Effect SO terms are shown with the most common first, logically grouped, followed by the less common terms sorted alphabetically.

    All Variants Table

    • Displays a summary of variant data for all annotated variants attached to a given allele.

    • Allele symbol

      above the table links to Allele detail page in MGI.
    • Genomic information contains Genome build, start coordinate, end coordinate, reference allele and variant allele when provided.
    • Transcript and Polypeptide information contains Accession ID, start coordinate, end coordinate, reference allele and variant allele when provided.
    • Curated data will be displayed if available, if no Curated data is available, Sourced data will be displayed.
    • Color coded boundaries around the fields distinguish between Curated(green) and Sourced(yellow).
    • Table is sorted by genome start, then transcript start.
    • Active row (by default the first row) will be highlighted with a pale yellow background; the full data for that variant is displayed in the main form.
    • The Variants table is read only (no editing).

    • Transcript IDs

      and

      Polypeptide IDs

      are hyperlinks to their relevant sites.
