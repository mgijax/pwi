P-WI Edit Variant functionality:

General Description

The Variant module is a search, edit, display page all in one. Each variant is related to a single allele, while an allele can have multiple variants. Detail for these variants may be viewed and edited one at a time (not all variants at once for a given allele).

Keyboard Short-cuts

There are reported issues using these short-cuts with Mac/Firefox.
  • Search = Ctrl/Alt S
  • Clear = Ctrl/Alt C
  • Modify = Ctrl/Alt M
  • Previous = Ctrl/Alt P
  • Next = Ctrl/Alt N
  • Associate Tag = Ctrl/Alt T

Color-coded Labels

  • Blue/Green = fields that may be searched (but not modified)
  • Green = fields that may be modified by the Curator
  • Blue = Read only fields
  • Yellow = Action buttons

Supported Browsers

  • Chrome and FireFox are supported on both PC and Mac computers.
  • Safari is NOT supported for the PWI. There are features that do not work.

Searching

All fields that have labels highlighted in blue/green and may be used for searching. They may be searched individually or in combination. At least 1 search field must be entered or an error message is displayed. When more than 1 field is used in a search, it is treated as an AND search. (i.e.- All entered fields must match the variants to be returned.)

Limit on # of results
  • The Variant module is currently configured to restrict the number of results to 1,001 variants. It may be preferable to use more search fields to reduce the number of results returned.
Wildcards
  • Wildcards may be used for most fields. Exceptions include: Accession IDs, Creation Date, and Modification Date.
  • A wildcard (%) may be used at the beginning and/or end of your search text. E.g. %paired% or casein%
Search by Accession IDs
  • Valid types of IDs: MGI Allele
  • A single or list of IDs may be entered. The IDs may be of one type or a mix.
  • Lists may be delimited by comma, space, or newline/carriage return
  • A list of IDs may be entered using Cut/Paste from an Excel file or QC report.
Search by Dates
  • The Creation and Modification Dates are time stamps held in the database.
  • Valid date formats for searching are:
    • mm/dd/yyyy or mm-dd-yyyy
    • mm/dd/yy or mm-dd-yy
    • yyyy/mm/dd or yyyy-mm-dd
  • Searching for a range of dates:
    • equal to: enter a single date
    • greater than: > mm/dd/yyyy
    • greater than or equal to: >= mm/dd/yyyy
    • less than: < mm/dd/yyyy
    • less than or equal to: <= mm/dd/yyyy
    • between two dates (inclusive): mm/dd/yyyy..mm/dd/yyyy

Search Results

When a user clicks on the Search button, the results are displayed in a table. Each result represents a single variant. The table displays the variants in alpha numeric order. The highlighted variant is the one that is displayed in detail. A user can highlight a different result using the Prev/Next buttons or clicking on a different row.

Detail Results

Detailed information is displayed in the Edit Tab for the single yellow highlighted variant in the list of variants returned in the Summary table. By default, the highlighted variant is the first one returned. A user may click on the Prev/Next buttons or use their mouse to select another variant to view. A user must be logged in to Modify a variant. When not logged the Modify button is greyed out.

Adding New Data

When Creating a New Allele Variant Record
  • If the user only provides Source info, then API will create both Source and Curated records and the Curated will be a duplicate of the Source info.
  • If the user provides both Source and Curated info, then the API will create both records using Source and Curated info.
  • This means the API will always create Source and Curated rows regardless of whether the user is changing the Curated version or just want to leave the Source and Curated the same.
